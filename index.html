Welcome to Rapido

Rapido helps you to write web documents with the only use of HTML snippets, it is designed with a focus on usability and frugality.

The present document has been created following Rapido guidelines and nothing more, you can easily get your own documents with the same layout and style. Rapido guidelines are described in the next sections.

The previous section shows a gap within the <article>...</article> tags, you can fill the gap with section snippets. Typically, sections have a heading and one or more paragraphs, but not only, as we will see sections may also contain figures, tables and much more.

Article header

Element <header> is used for the title and few introductory paragraphs of your document. Generally, it is placed as first element withing the <article> tag.

On the following, code snippets are supposed to be pasted inside element <article>, so that tags <main> and <article> are understood and can be omitted.

Side comments

This section describes how to add a side comment. Side comments typically show parenthetical content considered part of the main flow, like qualifying remarks directly related to the section.Hello, I am a side comment. I am quite good in telling anecdotes or cool details, read me if you want, you don't have to.

A side comment should be inserted withing the paragraph of a section, it will appear at the right of the paragraph and vertically aligned with it, or just below the paragraph on smaller screens.

Images and videos

Both images and videos are displayed within <figure>...</figure> tags, possibly supplied with a caption. Tag <figure> is supposed to be positioned near the paragraph where the image (or video) is mentioned. Tag <figcaption> can contain a heading and one or more paragraphs.

Exactly the same html snipped is used for displaying videos, with the obvious difference of using tag <video> in place of tag <img>, as shown below.

Quotations

This section shows how to add quotations into your document. Element <blockquote> can be inserted in a section withing other paragraphs and may contains one or more paragraphs as well.

Rapido is just that: copy and paste the html snippets and fill the gaps.

Quotations highlight pieces of information and they are styled with a font bigger than the sorrounding text. If appropriate, use the cite attribute for adding a URL of the quotation source and the <cite> element for the text representation, both of them are optional.

Tables

As well as with images and videos, tables also are displayed withing a figure and can be described by a caption. Tables are useful for displaying tabular data and they are disegned to visualise several columns: if the table is bigger than the window width the hidden part is visible through mouse scrolling.

Lists

As usual, lists are created with tags <ul> and <ol>, whereas <li> element is used to represent an item in a list. Each item may containg a <span> element and one or more <p> elements. The width of the two elements follow different rules:

  • ●

    Element <span> has a fixed width, use it for adding numbers (<span>1.</span>) or bulletts (<span>●</span>) in front of each item. Element <span> is optional.

  • ●

    Elements <p> also have a fixed width, with the exception of the last child which grows in width untill the remaining space is filled. Use at least one <p> element.

We take the opportunity of this example to describe the semantic meaning of elements <em>, <strong> and <mark>.

  1. <em>

    emphasis

    Use it for adding emphasis to a word, compared to surrounding text. Example: "Is this the queen of the questions?"

  2. <strong>

    importance

    Use it to mark text that has greater importance than surrounding text. Example: "Do a small walk every hour."

  3. <mark>

    relevance

    Use it to denote content which has a degree of relevance. Example: "Words that match a search operation will be highlighted".

Code

Code can be inserted in a document within a <figure> node, in the same way as with images and videos. Wrap the <code> element within a <pre> element to add multiple lines of code.

HTML code needs to be escaped, use string &lt; for < and string &gt; for >.

Inline code is also possible with the only use of the <code> tag.

Math formulas
